## 📁 Folder Structure

```
lib/
├── core/
│   ├── constants/       # App-wide constants (never hardcode strings)
│   ├── errors/          # Failure & Exception classes
│   ├── extensions/      # BuildContext, String, DateTime, Widget extensions
│   ├── network/         # Dio client + interceptors (auth, retry, logging, error)
│   ├── router/          # GoRouter config + route constants
│   ├── theme/           # Material 3 theme, colors, text styles
│   └── utils/           # Logger, validators, helpers
│
├── features/
│   └── <feature_name>/        # One folder per feature
│       ├── data/
│       │   ├── datasources/   # Remote & local data sources
│       │   ├── models/        # Freezed data models (JSON/Firestore serialization)
│       │   └── repositories/  # Concrete repository implementations
│       ├── domain/
│       │   ├── entities/      # Pure Dart domain entities
│       │   ├── repositories/  # Abstract repository contracts
│       │   └── usecases/      # Single-responsibility use cases
│       └── presentation/
│           ├── pages/         # Screen widgets
│           ├── providers/     # Riverpod providers & notifiers
│           └── widgets/       # Feature-specific widgets
│
├── shared/
│   ├── providers/       # App-level providers (theme, connectivity)
│   └── widgets/         # Reusable widgets (AppButton, AppTextField, etc.)
│
├── firebase_options.dart  # Generated by FlutterFire CLI
└── main.dart
```

## 🏃 Quick Start

### 1. Use this template

```bash
# Clone
git clone https://github.com/eng-kerolos-adel/flutter-riverpod-template.git my_app
cd my_app

# Rename the project (macOS/Linux)
find . -type f \( -name "*.dart" -o -name "*.yaml" -o -name "*.xml" \) \
  -exec sed -i '' 's/flutter_riverpod_template/my_app/g' {} +
```

### 2. Install dependencies & generate code

```bash
make setup
# or manually:
flutter pub get
dart run build_runner build --delete-conflicting-outputs
```

### 3. Configure Firebase

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Connect your Firebase project
flutterfire configure
```

This generates `lib/firebase_options.dart` with your real keys.

### 4. Set up environment config

```bash
cp .env.example.json .env.json
# Edit .env.json with your values
```

```json
{
  "BASE_URL": "https://api.yourapp.com/v1"
}
```

Run with: `flutter run --dart-define-from-file=.env.json`

### 5. Run the app

```bash
make run
```

## 🏗️ Architecture

### Clean Architecture — Feature-First

Each feature is self-contained with three layers:

```
feature/
  data/       ← Knows about Firebase, HTTP, local DB
  domain/     ← Pure Dart. Zero external dependencies.
  presentation/ ← Flutter widgets + Riverpod providers
```

**Data flows only inward:** `data → domain ← presentation`

### Error Handling with Either

Repository methods return `Either<Failure, T>` — no exceptions escaping to the UI:

```dart
// Domain layer
Future<Either<Failure, UserEntity>> signIn({...});

// Presentation layer
final result = await ref.read(authNotifierProvider.notifier).signIn(...);
result.fold(
  (failure) => context.showSnackBar(failure.message, isError: true),
  (user)    => context.go(AppRoutes.home),
);
```

### Riverpod Providers Pattern

```dart
// 1. Stream provider for real-time data
@riverpod
Stream<UserEntity?> authState(AuthStateRef ref) =>
    ref.watch(authRepositoryProvider).authStateChanges;

// 2. Notifier for actions with state
@riverpod
class AuthNotifier extends _$AuthNotifier {
  @override
  AsyncValue<void> build() => const AsyncData(null);

  Future<void> signIn({...}) async {
    state = const AsyncLoading();
    final result = await ...;
    state = result.fold(
      (f) => AsyncError(f, StackTrace.current),
      (_) => const AsyncData(null),
    );
  }
}
```

## 🔥 Firebase Setup

### Firestore Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
  }
}
```

### Remote Config Defaults

Set defaults in `remote_config_defaults.json` before calling `fetchAndActivate()`.

## 🧩 Adding a New Feature

1. **Create the folder structure:**

```bash
mkdir -p lib/features/my_feature/{data/{datasources,models,repositories},domain/{entities,repositories,usecases},presentation/{pages,providers,widgets}}
```

2. **Define the domain entity** (`domain/entities/my_entity.dart`)
3. **Write the repository contract** (`domain/repositories/my_repository.dart`)
4. **Create use cases** (`domain/usecases/`)
5. **Implement the data model** with Freezed (`data/models/`)
6. **Implement the repository** (`data/repositories/`)
7. **Create Riverpod providers** (`presentation/providers/`)
8. **Build the UI** (`presentation/pages/` + `presentation/widgets/`)
9. **Add routes** in `AppRoutes` and `app_router.dart`
10. **Run code gen:** `make gen`

## 📝 Code Generation

This template uses code generation for:

| Generator | What it generates |
|---|---|
| `riverpod_generator` | `@riverpod` annotations → providers |
| `freezed` | Immutable models, `copyWith`, pattern matching |
| `json_serializable` | `fromJson` / `toJson` for models |
| `retrofit_generator` | Type-safe REST API clients |

Always run after changing annotated files:
```bash
dart run build_runner build --delete-conflicting-outputs
```
